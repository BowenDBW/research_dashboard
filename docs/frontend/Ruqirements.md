## 需求分析
## 前端：
使用material ui 9.0（带MCP ai对话组件的）设计，react框架。开启后跳转至桌面端主页，主页跳转到其他一级页面通过
### 一级页面：桌面端主页
##### 可以隐藏的菜单栏
1. 上方显示爬虫脚本运行记录（最后一次爬取时间，文库内文章总量等信息），顶部标题可点击进入跳转至文章列表页（右侧有肩头提示）
2. 下方（占80%）显示新建对话按钮，显示一条条历史对话记录
3. 底部显示设置按钮，点击后跳转至设置界面（右侧有肩头提示），版本号在设置下方显示
##### 主页部分
1. 一个大模型对话框，用户可以输入问题，点击发送按钮后，显示大模型的回答
2. 一个切换到paper search的按钮，切换后显示搜索范围起止时间选择框（精确到天）大模型根据用户输入寻找符合要求的论文
3. 一个切换到文章逐章总结的按钮，弹可输入列表框，输入文章标题的部分并选择下拉列表里的可选项，点击发送按钮后，显示大模型对文章内容的总结
4. 文章列表页的总结按钮可以直接跳转到 3 的功能，且不需要再输入任何信息
##### 右边常驻工具栏
1. 收藏夹文件树，默认不展开（这个文件树参考收藏文章页，但不可删改，点击文件夹可展开，每个文件有其他地方反复叙述过的5个控件），顶部标题可点击进入（右侧有肩头提示）
2. 这周浏览文章的历史记录从最近往后推，顶部标题可点击进入（右侧有肩头提示）
3. 订阅设置，点击后弹出对话框，包含增删查订阅的关键词（用于文章列表页的筛选）和增删查订阅的作者（用于文章列表页的筛选）

### 一级页面：文章列表页
1. 可以根据时间段，来源（期刊或会议名称）可输入文字的多选下拉列表；作者名，字词匹配（文本搜索框）；领域（cs.LG, cs.AI等）可输入文字的多选下拉列表来筛选文章
2. 可展示文章列表，包含文章标题、作者、来源、发布时间等信息，可自定义一页展示的文章数量
3. 有4个功能按钮，分别查看文章摘要，arxiv或来源网页，收藏文章，下载文章PDF
4. 可以点击收藏文章
5. 可以点击AI按钮跳转到大模型页面逐章总结文章内容
6. 有返回主页按钮

### 一级页面：收藏文章列表页
顶部显示新建文件夹，上一级按钮，按钮下时文件夹路径，路径每一级点击可以回退；下面是一个混合文件夹和文件的列表
有返回主页按钮
##### 文件列表项
1. 可展示文章标题，最右侧有按钮可以展开文章详细信息包含文章标题、作者、来源。
2. 展开后，有4个功能按钮，分别查看文章摘要，arxiv或来源网页，收藏文章，下载文章PDF
3. 可以点击AI按钮跳转到大模型页面逐章总结文章内容
4. 可以右键跳出菜单点击取消收藏（需要弹窗确认）
##### 文件夹列表项
1. 可以右键修改文件夹
2. 可以右键删除文件夹（需要弹窗确认）
3. 可以右键剪贴文件夹
4. 可以单击进入该文件夹
##### 空白处
1. 可以右键新建文件夹
2. 可以右键粘贴文件夹

### 一级页面：设置界面
1. 可以设置文章爬取来源（arxiv, semantic scholar等）
2. 可以设置文章爬取的频率（几小时爬取一次）
3. 可以设置文章数据库的路径
4. 可以设置开机自启动
5. 可以设置选用云端大模型（填api和key）
6. 可以设置选用本地大模型（子选项为huggingface模型名或指定路径）
7. 有返回主页按钮

### 一级页面：阅读历史信息
1. 用户可以查看自己的阅读历史记录，按照时间顺序展示（用户点击任一功能按钮即视为阅读，但行为数据库可以分类）
2. 用户可以分类搜索，按时间搜索自己的阅读历史记录
3. 用户可以点击任一历史记录条目，跳转至文章列表页，并展示该文章的详细信息
4. 展开后有4个功能按钮，分别查看文章摘要，arxiv或来源网页，收藏文章，下载文章PDF

### 订阅子对话框
1. 增删查订阅的关键词（用于文章列表页的筛选）
2. 增删查订阅的作者（用于文章列表页的筛选）

下载pdf实际会跳转浏览器预览pdf，链接由后端提供
