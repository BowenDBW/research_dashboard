# Research Dashboard 需求分析文档

## 1. 项目概述

Research Dashboard（代号 Claw）是一个桌面端学术研究辅助工具，旨在帮助研究人员高效地发现、管理和阅读学术论文。通过 AI 能力，提供智能对话、论文推荐、文章总结等功能，提升学术研究效率。

### 1.1 目标用户

- 学术研究人员
- 研究生
- 科研机构工作人员

### 1.2 核心价值

- **智能发现**：AI 驱动的论文搜索与推荐
- **高效管理**：收藏夹、日报、订阅等多维度内容组织
- **深度阅读**：文章摘要弹窗、逐章总结、Markdown 日报阅读

## 2. 功能模块

### 2.1 AI 对话系统

主页提供三种 AI 对话模式，通过 Tab 切换：

| 模式 | 功能描述 |
|------|----------|
| AI 对话 | 自由问答，讨论学术问题、获取研究灵感 |
| AI 搜索推荐 | 自然语言论文搜索，支持日期范围筛选 |
| 文章总结 | 选择文章后生成逐章结构化总结 |

**交互规则**：
- 模式切换时自动创建新会话，不同模式间不共享对话历史
- 输入框居中显示，最大宽度 800px
- 每个模式有独立的空状态欢迎界面

### 2.2 多模型支持

支持同时配置多个云端和本地模型服务商：

**云端模型**：
- OpenAI、Anthropic、Google 等服务商
- 配置项：服务商名称、API 端点、API Key、模型列表
- 每个模型可设置显示名称

**本地模型**：
- **服务器模式**：Ollama 等本地推理服务，配置端点地址
- **MLX 模式**：Apple Silicon 设备专用，支持 HuggingFace ID 或本地路径

**模型选择器**：
- 显示模型名称 + 服务商名称（透明小号）
- 根据模型类型显示不同图标（云端/服务器/Apple）

### 2.3 文章爬取与管理

**数据源支持**：
- arXiv
- Semantic Scholar
- IEEE
- Springer

**文章属性**：
- 标题、作者、摘要、来源、发布日期
- PDF 链接、原文链接
- 研究领域标签

**文章操作**：
- 收藏/取消收藏
- 分享链接
- 导出引用
- 打开摘要弹窗

### 2.4 收藏夹系统

- 支持三层文件夹嵌套结构
- 文件夹图标（橙色）、文件图标（蓝色）
- 点击文件夹展开/收起
- 点击文章打开摘要弹窗

### 2.5 Google 推荐

基于用户订阅关键词和作者，自动从 Google Scholar 抓取的每日推荐文章：

**推荐内容**：
- 日期
- 推荐文章数量
- 文章列表（使用 ArticleCard 展示）

**交互**：
- 月份选择器筛选
- 分页浏览（每页 5/10/20 条可选）
- 点击卡片打开文章列表弹窗，可查看每篇文章详情

### 2.6 历史记录

分为两类历史：

| 类型 | 内容 |
|------|------|
| 阅读历史 | 浏览过的文章列表 |
| 对话历史 | AI 对话会话列表 |

**交互**：
- 右侧栏通过 ToggleButton 切换类型
- 点击历史项跳转到对应页面

### 2.7 订阅设置

管理关注的论文来源：

- 关键词订阅（如 "transformer"、"cs.LG"）
- 作者订阅（如 "Yann LeCun"）
- 通过订阅对话框进行增删改

## 3. 用户界面设计

### 3.1 整体布局

三栏式桌面布局：

```
┌─────────────┬───────────────────────────┬─────────────┐
│  左侧栏      │        中间内容区          │   右侧栏     │
│  280px      │       flex: 1             │   300px     │
│  可折叠      │                           │   可折叠     │
│             │                           │             │
│  会话列表    │      路由页面内容          │  Google 推荐 │
│  新对话按钮  │                           │  收藏夹      │
│  主题切换    │                           │  历史记录    │
│  设置按钮    │                           │  订阅设置    │
└─────────────┴───────────────────────────┴─────────────┘
```

### 3.2 主题支持

- 浅色主题 / 深色主题切换
- 主题状态持久化存储

### 3.3 右侧栏模块颜色

| 模块 | 图标颜色 |
|------|----------|
| Google 推荐 | 黄绿色 `#9CCC65`（内容）`#C5E1A5`（标题） |
| 收藏夹 | 淡蓝色 `#BBDEFB` |
| 历史记录 | 淡红色 `#FFCDD2` |
| 订阅设置 | 淡紫色 `#B39DDB` |

## 4. 技术要求

### 4.1 技术栈

| 层级 | 技术选型 |
|------|----------|
| 框架 | React 18 + TypeScript |
| UI 库 | Material-UI (MUI) v5 |
| 日期 | @mui/x-date-pickers + dayjs |
| 状态管理 | Zustand |
| 路由 | React Router v6 (HashRouter) |
| 桌面壳 | Tauri |

### 4.2 路由结构

```
/                    # 主页（AI 对话）
/articles            # 文章列表
/favorites           # 收藏夹
/favorites/:folderId # 收藏夹子目录
/history             # 历史记录
/daily               # Google 推荐
```

### 4.3 数据持久化

- 本地存储用户设置
- 数据库存储文章、收藏、历史等
- Tauri 提供本地文件系统访问

## 5. 非功能性需求

### 5.1 性能要求

- 首屏加载时间 < 3s
- 列表滚动流畅（虚拟滚动可选）
- 响应式交互反馈

### 5.2 可维护性

- 组件化开发，单一职责
- 类型安全的 TypeScript
- 状态集中管理

### 5.3 可扩展性

- 预留多语言支持接口
- 可配置的数据源
- 可插拔的模型服务商
